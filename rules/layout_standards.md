

> ## 前端 UI 生成规范 — 企业管理后台
>
> ### 核心原则
> 生成商业级 SaaS 产品质量的界面，所有色彩/字号/间距均使用项目设计令牌，不得硬编码任何样式值。
>
> ---
>
> ### 一、页面整体布局规范（应用框架结构）
>
> 整个应用的框架为 **左菜单 + 右内容** 的经典后台布局，右侧内容区再分为 **面包屑区 + 内容区** 两层。
>
> ```
> ┌──────────────────────────────────────────────────────────┐
> │  ┌──────────────┐  ┌──────────────────────────────────┐  │
> │  │   左侧菜单栏  │  │  面包屑 / 用户信息（固定在顶部）   │  │
> │  │   - logo     │  ├──────────────────────────────────┤  │
> │  │   - 折叠按钮 │  │                                  │  │
> │  │   - 导航菜单 │  │   内容区（填满剩余空间，可滚动）   │  │
> │  │              │  │                                  │  │
> │  │  固定宽度    │  │                                  │  │
> │  │  可折叠      │  │                                  │  │
> │  │  内部滚动    │  │                                  │  │
> │  └──────────────┘  └──────────────────────────────────┘  │
> └──────────────────────────────────────────────────────────┘
> ```
>
> #### 布局层级结构
>
> | 层级 | 区域名称 | 对应元素 | 定位方式 | 关键 CSS 约束 |
> |------|---------|---------|---------|-------------|
> | 第一层 | 应用整体 | `.app` | flex row | `height: 100vh; overflow: hidden` |
> | 第二层 | 左侧菜单栏 | `.layout-sidebar` | flex row 中的固定宽度子项 | `flex-shrink: 0; overflow-y: auto` |
> | 第二层 | 右侧主内容区 | `.layout-main` | flex row 中的弹性子项 | `flex: 1; display: flex; flex-direction: column; overflow: hidden`（自身禁止滚动） |
> | 第三层 | 面包屑区 | `.main-header` | 右侧主内容区的顶部固定项 | `flex-shrink: 0` |
> | 第三层 | 内容区 | `.main-content` | 右侧主内容区的弹性填充项 | `flex: 1; overflow-y: auto` |
>
> #### 左侧菜单栏（Sidebar）规范
>
> - 宽度使用设计令牌 `var(--sidebar-width-expanded)` / `var(--sidebar-width-collapsed)`
> - 折叠按钮位于菜单栏顶部右侧，点击切换展开/收起
> - 菜单项使用 `el-menu` 组件，配置 `router` 属性实现路由联动
> - 菜单栏内容较长时可跟随面包屑区整体纵向滚动（`overflow-y: auto`）
> - 收起状态下仅显示图标，需在菜单项中使用 `<el-icon>` + `<span>` 配合 `collapse` 属性
>
> #### 面包屑区规范
>
> - **始终固定在窗口顶部**，不随页面内容滚动
> - 左侧显示当前页面路径（面包屑），右侧显示用户/操作信息
> - 启用 `flex-shrink: 0` 保证高度不被压缩
> - 背景色使用 `var(--fill-surface)`，底部加 1px 分割线
>
> #### 内容区规范
>
> - **宽高都填充满父容器**：通过 `flex: 1` 占据主内容区剩余全部空间
> - **内部内容超出可滚动**：设置 `overflow-y: auto`，滚动行为局限在内容区内部
> - 内边距使用 `var(--spacing-md)`
> - 禁止在内容区外层容器设置 `overflow-y: auto`，避免面包屑区一起滚动
>

> ### 二、按页面类型选择布局（禁止千篇一律）
>
> | 页面类型 | 布局模式 |
> |------|------|
> | 数据列表页 | 筛选栏 + Table 表格 + 分页 |
> | 详情/Profile 页 | 头部信息卡 + Tab 分组 + 字段网格 |
> | 表单页 | 分组表单 + 字段说明 + 底部操作栏 |
> | 数据概览页 | 统计卡片 + 图表 + 列表组合 |
>
> ---
>
> ### 三、组件结构要求
>
> **列表页**
> - 筛选区：下拉框 + 输入框 + 查询按钮，右侧对齐新增等操作按钮
> - 表格：固定表头，状态列用 Badge 标签，进度用横向进度条，操作列固定右侧用图标按钮
> - 底部分页：显示总条数 + 页码
>
> **详情页**
> - 头部卡片：左侧图片/头像 + 名称 + 副信息，右侧 2~4 个核心统计数字带图标，竖线分隔
> - Tab 导航：下划线样式，分组承载不同维度内容
> - 字段区：三列网格，label 在上 value 在下
> - 关联信息：用背景色小卡片包裹区分
> - 顶部操作：返回链接在左，编辑/删除按钮在右
>
> **表单页**
> - 字段按业务语义分组，每组有标题
> - 必填字段标红星，字段下方可加说明文字
> - 操作按钮固定在底部或顶部右侧
>
> ---
>
> ### 四、交互状态（必须实现，不可省略）
>
> - 按钮：hover / active / disabled / loading
> - 输入框：focus 边框高亮
> - 表格行：hover 背景变化
> - 数据加载：骨架屏或 Spin，禁止空白等待
> - 空数据：展示空状态插图 + 提示文字
>
> ---
>
> ### 五、禁止事项
>
> - 禁止硬编码任何色值、字号、间距数值
> - 禁止用 emoji 替代图标，统一使用 @ant-design/icons 或 lucide-react
> - 禁止占位数据（如"2025-01-03"）出现在字段 label 位置
> - 禁止缺少任何交互状态
> - 禁止 Demo 级草稿代码，每次输出即可直接使用的质量
