### 1. 模块定位
实现**一键联动119、120、110**等应急部门，在突发事件发生时快速通知、协同指挥，缩短应急响应时间。支持**联动触发 → 通知送达 → 响应确认 → 事中跟踪 → 事后反馈**的闭环管理。一级页面突出**联动按钮状态、本次事件联动记录、各联动单位响应状态**，驱动值班人员快速启动联动、跟踪救援进展；二级页面支持联动历史查询、响应效率分析及联动复盘。

+ **使用人群**：镇政府应急指挥中心/值班人员 → 一级页面突出“一键联动”按钮和各救援单位响应状态，驱动快速启动应急联动。
+ **数据显示目的**：告诉使用者“当前是否有正在进行的联动、各单位是否已响应” → 设置“最近联动事件列表”，支持查看详情及跟进。
+ **统计口径**：一级页面默认**所有进行中的联动事件（状态为已触发、响应中、处置中）**，以及最近一次联动事件的摘要；二级页面支持按时间、事件类型、联动单位筛选全部历史联动记录。

### 2. 一级页面核心数据（主屏概览）
| 核心指标 | 统计口径 | 展示形式 | 目的与行动含义 |

| 进行中联动事件数 | 状态为“已触发”“响应中”“处置中”且尚未闭环的联动事件总数 | 数字（红色背景），标注“联动中” | 告知值班人员当前有未结束的联动，需持续跟踪 |

> **模块标题旁**：显示“联动状态实时刷新”小标签，并设“查看全部历史”链接跳转二级页面。

### 3. 二级专题页面核心数据（点击下钻后）
进入“救援指挥联动”专题页，提供**时间筛选器**（今日/近7天/近30天/自定义）、**事件类型筛选**（火灾/医疗急救/治安/交通事故/其他）、**联动单位筛选**（119/120/110），分为四大区块：

#### 3.1 综合统计看板（随筛选条件变化）
+ 联动事件总数
+ 已完成事件数 & 进行中事件数
+ 平均响应时长（从触发到首个单位响应确认的分钟数）
+ 平均到场时长（从触发到单位到达现场的分钟数，需反馈数据）
+ 各联动单位响应率（响应次数 / 被联动次数 ×100%）
+ 联动完成率（已闭环事件数 / 总事件数）

#### 3.2 联动事件列表（支持筛选、排序）
+ 表格字段：触发时间、事件类型、事件地点、联动单位（勾选情况）、各单位响应状态、总体状态（已触发/响应中/处置中/已闭环）、响应时长（分钟）
+ 支持按触发时间、响应时长、状态排序和筛选
+ 每条事件右侧可操作：**查看详情**（下钻到事件联动明细）、**重发通知**（如某单位未响应）

#### 3.3 单次联动事件明细页（点击事件详情后）
+ **事件基本信息**：
    - 触发时间、触发人（值班员）
    - 事件类型、事件地点（地图打点）
    - 事件描述（如“石楼村厂房火灾，可能有人被困”）
    - 附件（现场照片/视频，若有）
+ **联动发起记录**：
    - 联动的单位（119/120/110）
    - 通知方式（一键联动接口/电话/短信）
    - 通知内容摘要
+ **各单位响应跟踪**（时间轴）：
    - 119：通知时间 → 确认响应时间 → 预计到场时间 → 实际到场时间 → 处置状态
    - 120：同上
    - 110：同上
+ **事中指挥记录**：
    - 值班员与现场指挥的通讯记录（文本或录音链接）
    - 指令下发（如增派、疏散、医疗后送）
+ **事后反馈**：
    - 事件处置结果（如火灾扑灭、伤员获救）
    - 各联动单位反馈报告（可上传）
    - 复盘建议
+ **操作**：重发通知、添加指令、标记事件闭环、导出联动报告（PDF）

#### 3.4 历史趋势与效率分析（折线图/柱状图）
+ **按日/周展示**：
    - 联动事件数量趋势
    - 平均响应时长趋势（考核联动效率）
    - 平均到场时长趋势（考核救援效率）
+ **联动单位分析**：
    - 各单位平均响应时长对比
    - 各单位平均到场时长对比
+ **事件类型分析**：
    - 不同事件类型的联动频次
    - 不同事件类型的平均响应/到场时长
