### 1. 模块定位
对辖区内**各单位、企业、网格员通过App上报**的事件进行**分类 → 认领 → 进度跟踪 → 限时督办 → 超时自动提醒**的全流程管理，确保事件处置责任到人、闭环可溯。一级页面突出**待认领事件、超期未办结事件**以及**即将超期事件**，驱动值班人员分类派发、跟踪督办；二级页面支持流转效率分析、超时原因追溯及责任人绩效评估。

+ **使用人群**：镇政府值班人员/事件流转管理员 → 一级页面突出“待认领事件数”和“超期未办结数”，驱动快速分类派发、督办。
+ **数据显示目的**：告诉使用者“哪些事件需要分类认领、哪些已超期需督办” → 设置“最新待办事件列表”按紧急程度（超期 > 即将超期 > 待认领）排序，支持一键认领、催办。
+ **统计口径**：一级页面默认**所有未办结事件**中（不限时间），重点关注“待认领”“办理中（含即将超期/已超期）”；二级页面支持按上报时间、事件状态、责任单位筛选全部历史数据。

### 2. 一级页面核心数据（主屏概览）
| 核心指标 | 统计口径 | 展示形式 | 目的与行动含义 |
| --- | --- | --- | --- |
| 待认领事件数 | 状态为“待分类/待认领”的事件总数（不限时间） | 大号橙色数字，标注“需分类” | 告知值班人员有未分类认领的上报事件，需尽快手工分类并指派责任单位 |
| 超期未办结事件数 | 当前时间已超过办理时限（含整改期限），且状态尚未为“已办结”的事件总数 | 大号红色数字，标注“严重超期” | 最高优先级，需立即督办责任单位，避免事件恶化 |
| 即将超期事件数 | 办理时限在未来24小时内，且状态尚未为“已办结”的事件总数 | 橙色数字 + 时钟图标 | 提前预警，督促责任单位在期限内完成 |
| 平均处置时长（近7天办结事件） | 近7天内已办结事件从上报到办结的平均时长（小时） | 数字 + 趋势箭头 | 评估整体流转效率，对比考核标准 |
| 最新待办事件列表 | 按**紧急优先级**（超期 > 即将超期 > 待认领）展示前5条，每条显示：事件标题/类型、上报来源（单位/企业/网格员）、上报时间、办理时限（或剩余时间）、当前状态（待认领/办理中/超期） | 列表，每条右侧带“认领”或“催办”按钮 | 快速定位到最紧急的事件，一键认领或催办 |


> **模块标题旁**：显示“数据范围：未办结事件”小标签，并设“查看全部历史”链接跳转二级页面。

### 3. 二级专题页面核心数据（点击下钻后）
进入“分类流转督导”专题页，提供**时间筛选器**（今日/近7天/近30天/自定义）、**事件状态筛选**（待认领/办理中/已办结/超期未办结）、**上报来源筛选**（企业/单位/网格员）、**责任单位筛选**（各部门/村居），分为四大区块：

#### 3.1 综合统计看板（随筛选条件变化）
+ 事件总数
+ 待认领数 & 办理中数 & 已办结数 & 超期未办结数
+ 按期办结率 = 按时限办结数 / 应办结总数 ×100%（不含超期）
+ 平均处置时长（从上报到办结的小时数，仅统计已办结）
+ 超期事件平均超期天数
+ 各类事件类型数量分布（如市容环境/安全隐患/矛盾纠纷等，由分类标签决定）

#### 3.2 事件列表（支持筛选、排序）
+ 表格字段：上报时间、事件标题/类型、上报来源（单位/企业/网格员）、责任单位（已认领）、办理时限、当前状态（待认领/办理中/已办结/超期）、剩余/超期天数、处置人
+ 支持按办理时限、状态、上报时间排序和筛选
+ 每条事件右侧可操作：**查看详情**（下钻到事件明细）、**认领/派发**（待认领状态）、**催办**（处理中或超期状态）

#### 3.3 单条事件明细页（点击事件详情后）
+ **事件基本信息**：
    - 上报时间、上报人（姓名、单位、联系方式）
    - 事件标题、事件类型（分类标签）
    - 事件详细描述（文字）、现场照片/视频（附件）
    - 位置（地图打点）
+ **流转认领记录**：
    - 分类认领人、认领时间、指派的责任单位/负责人
    - 办理时限（根据配置规则自动计算或手动设定）
+ **处置跟踪**：
    - 处置过程记录（时间、操作人、处理内容、上传反馈）
    - 是否延期申请及审批记录
+ **督办记录**：
    - 自动提醒记录（超时提醒时间、方式）
    - 人工督办记录（督办人、时间、方式、回复）
+ **办结核实**：
    - 办结时间、办结人、办结说明、群众满意度评价（如有）
+ **操作**：重新分类认领、调整办理时限、发送督办通知、标记办结、导出事件档案（PDF/Excel）

#### 3.4 历史趋势与效率分析（折线图/柱状图）
+ **按日/周展示**：
    - 每日上报事件数 vs 办结事件数
    - 按期办结率趋势
    - 平均处置时长趋势
+ **效率分析**：
    - 各责任单位按期办结率排名
    - 各类事件的平均处置时长对比
    - 超期原因分析（分类错误/人手不足/流程复杂等，需标签统计）
+ **上报来源分析**：
    - 网格员 vs 企业 vs 单位的上报数量及办结率对比
