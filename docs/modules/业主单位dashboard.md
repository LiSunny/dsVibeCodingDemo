## 消防维保平台业主端：消防维保监管看板（针对库区一把手）

### 1. 模块定位
该模块面向库区最高管理者，提供**消防维保履约态势、重大风险、待办事项**的一览视图，解决“维保黑盒”问题。核心逻辑是：**以事件驱动，突出未闭环隐患、待确认维修、合同超期等关键动作**，形成“风险发现 → 派单 → 维修 → 确认 → 闭环”的可视化监管链。一级页面聚焦**近期（默认近7天）需要管理者关注或决策的高优先级事项**；二级页面支持历史追溯与统计分析。

+ **使用人群**：库区主任/总经理（一把手） → 一级页面突出“重大隐患超时未处理”“待确认维修”“合同即将到期”，驱动管理者立即督办。
+ **数据显示目的**：告诉使用者“哪里存在安全风险、哪些维保工作未完成、哪些需要我签字确认” → 设置“高优先级事项清单”带督办按钮。
+ **统计口径**：一级页面默认近7天（隐患发现时间、维修完成时间、合同到期时间在近7天内），超过7天的自动归入历史，但重大隐患若未闭环则持续显示；二级页面支持全部历史数据筛选。

### 2. 一级页面核心数据（主屏概览）

| 核心指标 | 统计口径 | 展示形式 | 目的与行动含义 |
| --- | --- | --- | --- |
| 未闭环重大隐患数 | 等级为“重大”且状态≠“已闭环”的隐患数（不限时间） | 大号红色数字，标注“需立即督办” | 告知管理者当前最严重的安全风险，驱动立即介入 |
| 待确认维修数 | 维修工单状态为“已完成，待甲方确认”且确认时限≤3天 | 橙色数字 + “需确认” | 提示哪些维修需要管理者安排现场验收或远程确认 |
| 合同剩余天数 | 当前维保合同距离到期日的天数 | 绿色（>30天）/橙色（≤30天）/红色（≤7天） | 提醒续签，避免服务中断 |
| 近7天维保任务完成率 | 近7天应完成维保任务数 / 实际按时完成数 ×100% | 环形进度条 + 百分比（≥90%绿，70-90%橙，<70%红） | 评估维保公司履约及时性 |
| 高优先级事项清单 | 近7天内发生的需要管理者关注的事项，按严重程度排序，最多5条（每条含事项类型、简要描述、发生时间） | 列表，每条右侧带“查看详情”或“督办”按钮 | 快速定位具体事项，一键下钻处理 |

> **模块标题旁**：显示“数据范围：近7天（重大隐患不限）”小标签，并设“查看全部历史”链接跳转二级页面。

### 3. 二级专题页面核心数据（点击下钻后）
进入“维保监管详情”专题页，提供**时间筛选器**（近7天/近30天/本年度/自定义/全部历史），分为四大区块：

#### 3.1 综合统计看板（随筛选时间变化）
+ 维保任务总数：时间范围内计划执行的维保任务数量
+ 按时完成率：按时完成的任务数 / 总任务数 ×100%
+ 隐患总数：时间范围内发现的隐患数量（按等级：重大/一般）
+ 隐患闭环率：已闭环隐患数 / 隐患总数 ×100%
+ 平均维修响应时长：从隐患发现到维修工单派出的平均小时数
+ 合同外维修费用总计：时间范围内产生的额外收费总额（需确认）

#### 3.2 隐患与维修清单（支持筛选、排序）
+ 表格字段：隐患等级、隐患描述、发现时间、关联设备、当前状态（未派单/维修中/待确认/已闭环）、响应时长、维修费用（合同外）
+ 支持按等级、状态筛选，按发现时间/响应时长排序
+ 每条记录右侧可操作：**查看详情**（下钻到该隐患的全流程时间轴）、**一键催办**（维修超时时可向维保公司发送催办消息）

#### 3.3 隐患详情页（点击隐患后）
+ 隐患基本信息：等级、发现时间、发现人（维保工程师）、设备位置、现场照片
+ 处理时间轴：发现 → 派单 → 维修开始 → 维修完成 → 测试 → 甲方确认（每一步时间、责任人、备注）
+ 维修记录：更换备件明细（名称、数量、单价、是否合同内），业主确认记录（签名/扫码截图）
+ 超时分析：是否超过合同约定的响应/修复时限，超时时长
+ 操作：若维修已完成待确认，显示“确认闭环”按钮；若超时未处理，显示“催促维保”按钮

#### 3.4 合同履约趋势分析（折线图/柱状图）
+ 按月展示：维保任务按时完成率、隐患闭环率、平均维修响应时长
+ 各季度合同外费用占比（合同内 vs 合同外）
+ 维保人员到场签到率（基于GPS签到数据）
