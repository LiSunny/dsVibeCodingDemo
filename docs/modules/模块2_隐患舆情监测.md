### 1. 模块定位
聚焦辖区内**未整改隐患**的实时监控与催办，重点跟踪整改期限临近或已超期的隐患，形成**发现 → 派发 → 整改 → 复核 → 闭环**的管理流程。一级页面突出近期（默认近7天）需紧急处理的隐患事项，驱动值班人员立即督办；二级页面支持历史隐患追踪、超期分析与整改效率评估。

+ **使用人群**：镇政府值班人员 → 一级页面突出“近7天需整改+超期”两类最紧急隐患，驱动立即督办。
+ **数据显示目的**：告诉使用者“现在催办哪些隐患” → 设置“最新未改隐患列表”按剩余时间排序，超期红色置顶。
+ **统计口径**：一级页面默认近7天（整改期限在7天内）+ 超期（不限时间），符合“紧急需要行动”原则；二级页面筛选器支持全部历史。

### 2. 一级页面核心数据（主屏概览）
| 核心指标 | 统计口径 | 展示形式 | 目的与行动含义 |
| --- | --- | --- | --- |
| 近7天需整改隐患数 | 整改期限在**未来7天内**（含今日）且**尚未整改完成**的隐患总数；**不包含已超期隐患**（超期隐患单独统计） | 大号橙色数字，标注“需督办” | 告知值班人员短期内必须跟进的隐患数量，驱动立即派发或催办整改责任单位 |
| 超期未整改隐患数 | 整改期限**已过**但尚未整改完成的隐患总数 | 大号红色数字，标注“严重超期” | 最高优先级，需立即上报或强制督办，防止隐患演变为事故 |
| 隐患整改率（近7天任务） | 近7天内应完成的隐患中，已整改完成的数量 / 近7天内应完成的隐患总数 ×100%（应完成指整改期限落在近7天内） | 环形进度条 + 百分比（≥90%绿，70-90%橙，<70%红） | 评估近期隐患整改工作的整体效率，判断是否存在普遍性拖延 |
| 高风险未改隐患数 | 隐患等级为“高”或“重大”且尚未整改完成的隐患总数（不限时间） | 红色数字 + 警示图标 | 突出最高风险点，即使整改期限较远也需重点监控，驱动提前干预 |
| 最新未改隐患列表 | 按**整改剩余时间升序**（超期 > 剩1天 > 剩2天…）展示前5条隐患，每条显示：隐患名称、责任单位/企业、整改期限、剩余天数（或“超期X天”） | 列表，每条右侧带“催办”按钮 | 快速定位最紧急的隐患，一键催办责任方 |


> **模块标题旁**：显示“数据范围：整改期限≤7天+超期”小标签，并设“查看全部历史”链接跳转二级页面。

### 3. 二级专题页面核心数据（点击下钻后）
进入“隐患舆情监测”专题页，提供**时间筛选器**（近7天/近30天/自定义/全部历史）及**状态筛选**（未整改/已整改/超期），分为四大区块：

#### 3.1 综合统计看板（随筛选时间及状态变化）
+ 隐患总数：时间范围内（按发现时间或整改期限，可配置）的隐患总量
+ 未整改数 & 已整改数 & 超期未改数
+ 整体整改率 = 已整改数 / 隐患总数 ×100%
+ 超期率 = 超期未改数 / 未整改总数 ×100%
+ 平均整改时长（从发现到整改完成的天数，仅统计已整改）
+ 高风险隐患占比（高风险数 / 隐患总数）

#### 3.2 隐患列表（支持筛选、排序）
+ 表格字段：隐患名称、发现时间、整改期限、责任单位/企业、隐患等级（高/中/低）、状态（未整改/已整改/超期）、剩余天数/超期天数、整改负责人
+ 支持按整改期限、剩余天数、等级、状态排序和筛选
+ 每条隐患右侧可操作：**查看详情**（下钻到隐患明细）、**一键催办**（发送提醒给责任单位）

#### 3.3 单条隐患明细页（点击隐患详情后）
+ **基本信息**：隐患描述、现场照片/视频、发现来源（巡查/上报/AI预警）、发现人、发现时间
+ **整改信息**：整改要求、整改期限、责任单位/企业、整改负责人及联系方式
+ **整改进度**：整改记录（提交整改照片、整改说明、复核人、复核结果）
+ **超期记录**：超期天数、催办历史（催办时间、方式、接收人、回复情况）
+ **操作**：重新派发、延期申请（需审批）、强制督办（上报上级）、标记完成（需上传整改证明）
+ **导出**：导出隐患档案为PDF/Excel

#### 3.4 历史趋势与效率分析（折线图/柱状图）
+ **按月/周展示**：
    - 新增隐患数 vs 整改完成数（堆积柱状图）
    - 整改率变化趋势
    - 超期未改数变化
+ **责任单位排名**：
    - 各企业/单位的未整改数排行（倒序，突出整改滞后单位）
    - 各企业/单位的平均整改时长对比
+ **隐患类型分析**：
    - 按隐患类型（消防/安全/环保/其他）的未整改分布
    - 不同等级的隐患整改率对比（高风险是否优先整改）
