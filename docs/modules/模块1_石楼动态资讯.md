### 1. 模块定位
政府向企业/目标单位发布通知、政策，并要求“已读确认”及反馈信息，形成**下发 → 阅读 → 反馈 → 催办**的闭环。一级页面聚焦近期（默认近7天）需处理的未读与反馈事项，驱动值班人员立即催办；二级页面支持历史追踪与统计分析。

+ **使用人群**：镇政府值班人员 → 一级页面突出“近7天未读企业”和“待反馈通知”，驱动立即催办。
+ **数据显示目的**：告诉使用者“哪些通知还未读、哪些需要补反馈” → 设置“最新未读通知列表”带催办按钮。
+ **统计口径**：一级页面默认近7天（通知发布时间在7天内），超过7天的通知自动归档，不再纳入“未读”催办范围；二级页面筛选器支持全部历史。

### 2. 一级页面核心数据（主屏概览）
| 核心指标 | 统计口径 | 展示形式 | 目的与行动含义 |
| --- | --- | --- | --- |
| 近7天未读企业数 | 近7天内发布的通知中，尚未点击“已读”的企业总数（去重企业-通知组合） | 大号红色数字，标注“需催办” | 告知值班人员当前需催办的企业数量，驱动立即催办 |
| 近7天整体已读率 | 近7天通知的总已读数 / 总应读数 ×100% | 环形进度条 + 百分比（≥90%绿，70-90%橙，<70%红） | 评估近期信息传达效率，宏观判断是否有普遍触达问题 |
| 待反馈通知数 | 近7天内，要求企业填写反馈信息但存在“已读未填”情况的通知数量 | 橙色数字 + “需提醒”图标 | 提示哪些通知需要二次提醒企业补充反馈 |
| 最新未读通知列表 | 近7天内有未读企业的通知，按发布时间倒序展示前5条（每条显示：标题、未读企业数、发布时间） | 列表，每条右侧带“催办”按钮 | 快速定位到具体通知，一键执行催办操作 |


> **模块标题旁**：显示“数据范围：近7天”小标签，并设“查看全部历史”链接跳转二级页面。

### 3. 二级专题页面核心数据（点击下钻后）
进入“石楼动态资讯”专题页，提供**时间筛选器**（近7天/近30天/自定义/全部历史），分为四大区块：

#### 3.1 综合统计看板（随筛选时间变化）
+ 通知总数：时间范围内发布的通知数量
+ 应读总人次：∑(通知的目标企业数)
+ 已读总人次：∑(已读企业数)
+ 整体已读率 = 已读总人次 / 应读总人次 ×100%
+ 未读总人次（需催办总量）
+ 要求反馈的通知数 & 反馈完成率（已填反馈企业数 / 应填反馈企业数）

#### 3.2 通知列表（支持筛选、排序）
+ 表格字段：通知标题、发布时间、发布人、目标企业数、已读数、未读数、已读率、是否需要反馈、反馈完成率
+ 支持按未读数/已读率排序，支持按发布人筛选
+ 每条通知右侧可操作：**查看详情**（下钻到该通知的明细）、**一键催办**（提醒所有未读企业）

#### 3.3 单条通知明细页（点击通知详情后）
+ 通知全文内容（附件下载）
+ 目标企业列表（表格）：企业名称、是否已读、已读时间、是否填写反馈、反馈内容
+ 未读企业清单（突出显示，支持批量催办）
+ 已读未反馈企业清单（突出显示，支持批量提醒填写）
+ 导出功能（导出企业明细为Excel）

#### 3.4 历史趋势分析（折线图/柱状图）
+ 按月/周展示：通知发布数量、平均已读率、未读企业数变化
+ 各企业历史已读率排名（用于年度考核）
+ 各类通知的已读率对比（例如：安全类通知 vs 政策类通知）
